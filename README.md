# スクラム開発チーム向け実践ガイド

このリポジトリは、スクラム開発チームが効果的にプロジェクトを進めるための包括的な実践ガイドを提供します。

## 📚 ガイド構成

### 1. [スクラム基礎](./docs/01-scrum-basics.md)
- スクラムの基本概念と価値
- スクラムチームの役割と責任
- スクラムイベントの詳細

### 2. [スプリント計画と実行](./docs/02-sprint-planning.md)
- スプリント計画会議の進め方
- バックログの作成と管理
- スプリント実行中のベストプラクティス

### 3. [デイリースクラムと進捗管理](./docs/03-daily-scrum.md)
- デイリースクラムの効果的な実施方法
- 進捗の可視化と追跡
- 障害の早期発見と解決

### 4. [スプリントレビューとレトロスペクティブ](./docs/04-sprint-review.md)
- スプリントレビューの準備と実施
- レトロスペクティブの効果的な進め方
- 継続的改善の実現

### 5. [プロダクトバックログ管理](./docs/05-product-backlog.md)
- ユーザーストーリーの作成
- 優先順位付けの方法
- 見積もりとリファインメント

### 6. [スクラムツールとテンプレート](./docs/06-tools-templates.md)
- 推奨ツールと設定
- テンプレート集
- 自動化と効率化

### 7. [トラブルシューティング](./docs/07-troubleshooting.md)
- よくある問題と解決策
- チームパフォーマンスの改善
- スクラム導入の成功要因

## 🚀 クイックスタート

1. [スクラム基礎](./docs/01-scrum-basics.md)から読み始める
2. チームの状況に応じて関連するガイドを参照
3. [テンプレート](./templates/)を活用して実践
